Initial release: managed redirects for Payload with a cache-backed, framework-agnostic middleware matcher. An orderable redirects collection (simple by default, with matching/query/metadata options behind a "Show advanced settings" toggle) supports internal-document or custom-URL destinations, five match types — exact, starts with, ends with, contains, and safe-validated regex with capture-group substitution — plus per-redirect case-insensitivity, query forwarding, an enabled toggle, and editor notes. `from` values are canonicalized (trailing slashes, sorted query, percent-encoding), redirect loops are rejected at save time and exact chains flattened at build time so visitors take a single hop, and open-redirect escapes are blocked. Hit tracking uses adapter-agnostic optimistic concurrency; endpoints can be locked down with a shared `secret`; localization builds one cache per locale. Ships a Next.js middleware (`onRedirect`, `debug`, `cacheMemoMs`, `secret`) and a pure `resolveRedirect` for Express/Hono, pluggable caches (Vercel Runtime Cache, Vercel Edge Config, Redis/Upstash, Cloudflare KV, JSON file, in-memory), destination-collection re-sync on move/delete, `select` narrowing, and a `migrateFromOfficialRedirects` helper for `@payloadcms/plugin-redirects`.
